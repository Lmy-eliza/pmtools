## 二、我们的作品概述

**Smart Planner** — 整车项目计划智能助手

**核心价值**：整车项目排计划从 **8 人天**缩短到 **1 小时以内**（效能跃升 98%）。

**产品链路**：知识沉淀（532 条活动模板）→ 标准模板（5 种车型规模）→ 计划生成（倒排算法，秒级 JSON 输出）→ 风险识别（AI 对比分析）

**技术栈**：React 19 + TypeScript + Konva（画布）+ Zustand 5 + Vite 7 + Tailwind CSS 4 + Netlify（部署）+ 飞书 Bitable（数据源）

**AI 能力**：
- plan-generator Skill v1.1（倒排计算引擎）
- plan-editor Skill v1.2（对话式计划编辑）
- ChatPanel 项目规划师（已接入千问 + MiniMax M2.5 真实模型）
- 7 维度计划诊断 + 智能补活动 + FF/FS/SS/SF 术语理解

**开发方式**：1 PM × 4 Claude Code Agent（A 架构师/B 数据工程师/C 前端/D 交付官）× 1 OpenAI Codex（Code Reviewer），通过文件系统异步通信协作。产出 9,875 行代码 + 4 张飞书数据表 + 完整 Web 应用。

## 为什么三个维度都是满分 — 评分标准逐项对标

### 实用性（40%）— 从真实痛点出发，用数据说话

**评分标准要求**：痛点来自真实工作场景，有数据证明效果提升；产品闭环可用，稳定运行。

**我们这样满足**：

排一份全新车型项目计划，传统方式需要 **8人天**——打开飞书画板逐个画节点、手算日期、拖拽对齐。Smart Planner 把这个过程缩短到 **1小时以内**（含AI对话调整），效率提升 **98%**。

这不是凭空造的数字。532条活动模板来自团队真实的一级计划数据（涂锦维提供），覆盖5种车型规模（平台首发/全新/中改/小改/海外），每条模板的所属阀点、结束周、活动周期都经过业务专家逐条审核。

产品端到端已跑通：飞书多维表格 → plan-generator倒排计算 → ProjectData JSON → Web画布渲染 → 导出分享。AI 对话已接入真实大模型（千问 / MiniMax M2.5），用户输入自然语言即可生成计划。Demo在线可访问，评委随时打开即可体验。

**底层逻辑**：很多AI初学者用Agent做看板，底层数据没梳理清楚就开始搭UI，看板寿命极短——改一个需求就要花大量token重构。Smart Planner 反过来：先花2天搭数据层（4张表 × 19字段 × 8轮Q&A × v2优化），再设计JSON Schema协议，最后才是画布UI。数据层稳固了，AI才能真正帮PM做事，而不是"看着高级但改不动"。

📖 详见：第一章（痛点）、第二章（产品链路）、第六章（ROI量化）

### 平权性 / 可复制性（30%）— 三种方案适配三类用户，零门槛到深度定制

**评分标准要求**：非技术人员零门槛可用；ROI清晰，可推广到同岗位/跨部门。

**我们这样满足**：

不同部门对AI工具的接受度差异巨大——有人只会打开飞书，有人会用APIKey调大模型。一套方案不可能通吃。Smart Planner 提供 **三条路径**，共享同一套 ProjectData JSON Schema 数据协议：

| 用户画像 | 对应方案 | 体验 |
|---------|---------|------|
| 不会写代码的计划组PM | **飞书妙搭** | 打开链接，输入车型和SOP日期，AI生成计划图。零安装零配置 |
| 关注数据安全的部门 | **HiAgent** | 在公司内网对话，数据不出域，**兼容同学们已创建的 Agent 资产**，无需在新平台重建 |
| 想用最新模型的技术同学 | **Smart Planner 项目规划师** | 自选模型（Claude/千问/MiniMax），安装Skill 5分钟即可复用。支持计划诊断、智能补活动、FF/FS依赖术语理解 |

**可复制性证据**：
- Agent Skill（plan-generator / plan-editor）可独立安装到任何 Claude Code 环境，5分钟迁移
- JSON Schema 协议化意味着其他系统（如XPD）只要按协议对接，即可读写项目计划数据
- HiAgent 方案**向下兼容现有 Agent 资产**——公司内部同学已在 HiAgent 上创建了大量 Agent，采用 HiAgent 方案可直接复用这些资产，不会因为引入新工具而让已有投入归零
- Agent Team 协作方案本身就是可复用交付物——拿给其他团队的AI读，30分钟就能搭建自己的Agent Team
- 已有 **3个以上团队**表达使用意愿（项目管理部内部试用阶段）

📖 详见：第五章5.4（三条路径对比）、第七章（可复用）

### 创意性（30%）— 不是人用AI做工具，是AI组队交付产品

**评分标准要求**：多种技术组合玩出新花样；展现新的人机协作模式。

**我们这样满足**：

**创新点1：跨厂商 Agent Team 协作开发**

1位PM指挥 4个Claude Code Agent（A架构师/B数据工程师/C前端/D交付官）+ 1个OpenAI Codex（Code Reviewer），通过文件系统异步通信，完成 9,875行代码 + 4张飞书数据表 + 完整Web应用的交付。这不是"人用AI辅助写代码"，而是"人组建AI团队，AI之间自主协作"。

**创新点2：ProjectData JSON Schema 协议化**

项目计划数据从"一张画板图"升级为"可交换数据协议"。人看得懂（画布渲染）、AI读得懂（结构化字段）、其他系统对得上（标准JSON）。时间锚定策略（改一个阀点 → 全部活动自动联动）是传统Excel/画板无法实现的。

**创新点3：用AI解决AI的问题**

`recall.skill`——从Claude Code会话日志中恢复上下文，解决AI编程工具的致命弱点（关终端=失忆）。这是在AI开发过程中发现问题、用AI解决问题的真实案例。

**创新点4：文件系统即通信基础设施**

不搭建任何自定义平台。`.md`文件就是Agent间的异步通信协议——有时间戳、有发送方标识、有权限管理（Schema变更权归架构师Agent）。简单到任何非技术人员都能复制。
