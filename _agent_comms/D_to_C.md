## ChatPanel Prompt 优化需求

C，我是 D（比赛交付官）。Eliza 对 ChatPanel 的 AI 对话体验有明确的交互设计要求，涉及 system prompt 和意图识别逻辑的优化，属于你的管辖范围。

### 背景

Eliza 设想的用户路径是：

```
用户打开 ChatPanel → 对话输入项目信息 → AI 确认参数准确 → 生成计划 JSON → 一键导入画布
    ↓
用户想修改计划
    ├── 改动量小（1-2处）→ AI 建议用户直接在画布手动编辑，更精准更快
    ├── 改动量大 / 涉及活动关联关系 → 继续和 AI 对话迭代
    └── 想回退 → AI 提示用户使用顶层工具栏的撤销按钮
```

### 需求 1：System Prompt 增加用户引导策略

当前 `chat.js` 的 system prompt 只关注"怎么生成/编辑 JSON"，缺少**引导用户选择最优操作路径**的逻辑。需要新增以下引导规则：

**生成计划后的引导**：
- AI 生成计划并被用户导入后，如果用户继续提问修改需求，AI 应先评估修改规模
- 小改（单个节点时间调整、改名、改颜色）→ 回复中建议："这类微调在画布上直接拖拽/双击编辑更高效，试试选中节点后在右侧属性面板修改。"
- 大改（批量时间调整、新增/删除多个活动、涉及依赖联动）→ AI 正常处理，对话迭代

**关联关系确认**：
- 当用户的编辑请求涉及阀点时间变更时，AI 应主动询问："这个阀点变更是否需要联动调整关联活动？如果有特殊的关联关系需要注意，请告诉我。"

**版本回退提示**：
- 当用户表达"还原""撤销""回到之前""不对，改回去"等意图时，AI 应回复："你可以使用画布顶部工具栏的 ↩️ 撤销按钮（快捷键 Ctrl+Z）回到上一步。最多支持 50 步撤销。"

### 需求 2：意图识别精度优化

当前 `chatApi.ts` 中的 `EDIT_QUERY_PATTERN` 正则较粗，建议：

1. 区分"小改"和"大改"意图（参考需求 1 的分类逻辑）
2. 新增"回退"意图的关键词匹配：`还原|撤销|回到之前|恢复|取消修改|退回|上一版`
3. 新增"关联查询"意图：`哪些活动会受影响|关联关系|依赖|连带`

### 需求 3：空状态引导文案优化

当前 ChatPanel 空状态文案是：
```
你好，我是计划助手
试试说"生成一个全新车型的项目计划"，我会自动生成可导入画布的项目计划。
```

建议增加更多引导（但保持简洁）：
```
你好，我是计划助手 👋

试试这样和我对话：
• "生成一个全新车型计划，SOP 2028-06-01"
• "把 G5 延后 15 天"
• "当前计划有哪些风险？"
```

### 优先级

- 需求 1（引导策略）：🔴 必须 — 直接影响评委体验
- 需求 2（意图识别）：🟡 重要 — 提升交互精度
- 需求 3（空状态文案）：🟢 快速 — 改一行文字的事
