# 产品规范

## 基本要求
- **页面设计**：简洁、功能清晰、易上手；页面角落需有使用指南和快捷操作指南（如有）；
- **Agent 兼容**：主动增加 JSON 格式导入导出功能，便于 Agent 直接读取和编辑；
- **项目存放**：各项目独立文件夹，保存至 `D:\claude learning\`；
- **参考标准**：拓展功能参考市场最优产品，必要时跨行业借鉴并创新。

---

## 页面设计规范
- **视觉**：喜欢漂亮、整洁的设计，发现好的 emoji 或视觉素材主动记录至记忆宫殿；
- **用词**：产品面向全行业，用词必须精准，不随意用口语化表达；
- **功能入口**：核心功能优先级高，次要功能收折或放置角落；
- **错误提示**：所有用户操作需有明确反馈，不静默失败。

---

## JSON 导入导出规范
- 所有结构化数据（计划、任务、配置）均支持 JSON 导出；
- 导出格式需语义清晰，字段命名用英文小写+下划线；
- 导入时做格式校验，错误给出明确提示而非直接崩溃；
- 示例参考：`D:\claude learning\project-planner`。

---

## 开发前置检查清单
在开始开发前，与用户逐项确认：
1. **核心需求**：根本问题是什么？
2. **目标用户**：谁在用？使用场景是？
3. **功能边界**：MVP 必须有什么？哪些是后续迭代？
4. **数据结构**：输入输出是什么格式？
5. **集成需求**：是否需要 Agent 调用 / JSON 接口？
