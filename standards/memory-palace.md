|用途|路径|
|-|-|

\---

## 目录结构

```
E:\\claude test data\\
├──对话记录-{项目名}\\
│   ├── plan-\*.md          # 计划文档
│   ├── context-\*.md       # 对话存档
│   └── README.md          # 项目摘要（技术栈、当前状态、关键决策）
├── knowledge\\
│   ├── design-patterns\\   # 好的设计参考
│   ├── emoji-resources\\   # emoji 资源
│   └── learnings\\         # 经验总结
└── project-index.md       # 项目索引（所有项目列表 + 状态）
```

\---

## 触发规则

|触发词|动作|
|-|-|
|「回顾上次进度」「继续XX项目」|读取对应项目 README.md + 最新 plan 文档|
|「存档」或大任务完成后|保存对话摘要到 `context-{日期}-{描述}-v{N}.md`|
|Plan 清空前|保存计划到 `plan-{日期}-{描述}-v{N}.md`|
|每次执行前|保存当前对话到 `context-{日期}-{描述}-v{N}.md`|
|阶段性任务完成后|主动更新项目 README.md|
|新项目启动或状态变更|更新 `project-index.md`|

\---

## 版本管理规范

* **版本号**：自然数递增，v1、v2、v3…（不用 v1.0 格式）；
* **命名规则**：`plan-{日期}-{描述}-v{N}.md`，示例：`plan-20260318-热力图增强-v1.md`；
* **保留策略**：每次生成新计划保留旧版本，不覆盖，新计划用新版本号；
* **锁板规则**：产品开发完成后，基于过往计划形成最终完整计划，命名 `plan-{日期}-completed.md`，锁板后不再修改；
* **迭代保留**：所有迭代版本保留，直到确认最终验收。

## 项目上下文管理

不在 CLAUDE.md 中维护「活跃项目」列表，通过 `project-index.md` + 各项目 `README.md` 管理上下文，用户提及项目名时读取对应文件。
