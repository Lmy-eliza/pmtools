# Smart Planner 多维表格数据库字段需求

> 用途：作为整车项目计划生成的底层数据源，AI Agent 读取表格数据 → 倒排生成 ProjectData JSON → 导入 Smart Planner 画布可视化
> 提出方：CC（Plan Generator Skill 开发侧）
> 日期：2026-05-25

---

## 背景

Smart Planner 的计划生成 Skill 需要从多维表格读取标准活动模板，基于用户输入的项目名称、车型规模、SOP日期，自动倒排生成完整项目计划。当前模板数据硬编码在 Markdown 文件中，需要迁移到多维表格作为结构化数据源。

**核心流程**：多维表格（活动模板库）→ Agent 读取 + 倒排计算 → ProjectData JSON → Web 画布渲染

---

### 依赖类型说明

| 类型 | 全称 | 含义 | 示例 |
|------|------|------|------|
| FS | Finish-to-Start | A做完才能开始B | 模具开发完成 → 试制开始 |
| SS | Start-to-Start | A开始后B也可以开始 | 设计评审开始 → 采购预研同步开始 |
| FF | Finish-to-Finish | A和B需要同时完成 | DV试验完成 → DVP报告同步完成 |
| SF | Start-to-Finish | A开始时B必须完成 | 很少用 |

### 节点类型对照

| 节点类型值 | 画布形状 | 用途 |
|-----------|---------|------|
| `rectangle` | 矩形（有宽度） | 开发活动、测试活动（有工期） |
| `diamond` | 菱形 | 阶段门TG、工程样件EP、验证DV/PV、里程碑 |
| `pentagon` | 五边形 | 阀门G0-G10 |
| `triangle` | 三角形 | 决策点 |
| `star` | 星形 | 重要里程碑 |
| `circle` | 圆形 | 状态节点 |

### 「是否必选」使用场景

- **必选**：该规模项目必须包含此活动，Agent 生成时不可跳过
- **可选**：默认包含，但用户说"精简版"时可以裁剪
- **按需**：用户明确要求时才加入

## Agent 怎么用这些数据

```
用户输入：项目名=P7+，规模=全新，SOP=2028-06-01

Agent 处理流程：
1. 读阀点定义表 → 取「全新」列的月数 → 从SOP倒推各阀点绝对日期
2. 读甬道定义表 → 筛选「适用项目规模」含「全新」的甬道 → 生成swimlanes[]
3. 读活动总表 → 筛选「适用项目规模」含「全新」且「是否必选」≠按需的活动
4. 每条活动：绝对开始日期 = 阀点绝对日期 + 开始周×7天
5. 读「前置活动」字段 → 生成 connections[]（活动间依赖线）
6. 读「是否关键路径」→ 标记 connection.isCriticalPath
7. 输出 ProjectData JSON → 用户粘贴导入 Smart Planner 画布
```
